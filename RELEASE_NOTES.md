# Release Notes for Google Play Store

## Version 1.0.1 - Release Notes (en-GB)

### What's New

**Enhanced Product Listing Experience**
- Added location search functionality - you can now search for addresses when creating product listings instead of only using the map
- Improved product listing review page - you can now edit product descriptions and pickup locations before final submission
- Fixed duplicate product cards appearing in chat conversations

**Improved Chat & Deal Management**
- Streamlined deal confirmation process - prices are now locked at the exact quoted amount for better security
- Enhanced "Mark as Sold" flow - choose whether products were sold within Junction or outside, with easy buyer selection
- Better rating system - sellers can now rate buyers after completing a sale
- Improved error messages throughout the app for a smoother experience

**Bug Fixes & Performance**
- Fixed various UI issues and improved app stability
- Enhanced error handling for better user experience
- Optimised app performance and reliability

**Admin Dashboard** (Backend)
- New dashboard statistics for administrators
- Improved user onboarding management

## Recommendations for Play Store

1. **Use the "Even Shorter Version"** (200 characters) for the "What's New" section in Play Console
2. **Keep it user-friendly** - Avoid technical jargon
3. **Focus on benefits** - What does this mean for users?
4. **Be concise** - Users scan quickly
5. **Update regularly** - Each new version should have fresh release notes

---

## Notes

- Play Store typically allows 200-500 characters for release notes
- You can provide different versions for different locales
- Release notes are visible to users before updating
- Keep previous versions' notes in mind for context
